# 讓「家裡電腦」也能編輯這個專案

> 適用情境:公司/家裡兩台電腦都想請 Claude 幫忙改程式。
> 推薦用 **GitHub Desktop**(圖形介面,不用打指令,最適合兩台電腦切換)。

---

## 一、家裡電腦第一次設定(只做一次)

1. **安裝 GitHub Desktop**
   到 https://desktop.github.com 下載安裝(Windows 版),一路 Next 裝完。

2. **登入你的 GitHub 帳號**
   開啟 GitHub Desktop → File → Options → Accounts → Sign in,
   用帳號 `david26984741-cell` 登入(會跳瀏覽器授權,按同意)。

3. **把專案抓下來(Clone)**
   File → **Clone repository** → 上方切到「GitHub.com」分頁 →
   清單裡點選 **`daily-postmarket`** →
   下方「Local path」選一個存放位置(例如 `文件` 資料夾)→ 按 **Clone**。
   完成後,家裡電腦就有一份完整專案了。

4. **在 Cowork(Claude)裡選這個資料夾**
   打開 Cowork → 選擇工作資料夾 → 指到剛剛 clone 下來的
   `daily-postmarket` 資料夾。這樣我在家也能直接幫你改程式了。

---

## 二、之後每次「在家要動手改」的標準流程

1. **先抓最新**:打開 GitHub Desktop → 按上方 **「Fetch origin / Pull origin」**。
   (很重要!因為每天機器人會自動更新資料,先抓下來才不會衝突。)
2. 請 Claude 幫你改程式 / 或自己改。
3. **上傳**:回到 GitHub Desktop,左下角填一句變更說明 →
   按 **「Commit to main」** → 再按右上 **「Push origin」**。
4. 幾分鐘後網站就會套用你的變更。

> 回到公司電腦時,一樣**先按 Pull** 再開始改,兩台就不會打架。

---

## 三、如果只是想「看網站 / 手動更新資料」

這些**完全不需要**上面的設定,任何電腦用瀏覽器就能做:
- 看網站:https://david26984741-cell.github.io/daily-postmarket/
- 手動更新:登入 github.com → Actions → Run workflow(詳見「操作小抄」)
